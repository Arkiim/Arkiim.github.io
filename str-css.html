Partie 3 : CSS

Structure des déclarations CSS

En premier lieu, avant de se lancer dans des explications superflues, il est nécessaire d'expliquer comment est organisé le css, comment fonctionnent les styles, les propriétés etc...

Comme l'on peut le voir sur la photo, le css est organisé de la manière suivante : le code est scindé en différentes "déclarations" css.

La déclaration est composé d'un ou plusieurs sélecteur(s), dont on parlera plus en détails plus tard, suivi d'une accolade ouvrante et une accolade fermante, exactement comme l'html avec les balises. A ce sélecteur, à l'intérieur de ces accolades, on applique une ou plusieurs propriété(s), comme par exemple, "font-family" qui va modifier la font ou encore "font-weight" qui va rend un texte plus gras, plus épais ou au contraire plus fin. La propriété css se termine toujours par ":" .

Structure d'une déclaration css

Pour pouvoir contrôler toutes ces propriétés, il faut les quantifier, leur donner une valeur. Ici la valeur de la propriété font-family est "Lato" mais elle en a une deuxième, en effet une propriété peut avoir plusieurs valeurs. Les valeurs secondaires, tertiaires ... de la propriété "font-family", ici "Arial", sont appelés des "fallback fonts" elles servent de police de secours dans le cas où la première serait absente sur l'ordinateur de l'utilisateur et donc indisponible.

Pour le font-weight quant à lui, sa valeur est de 900, 400 correspond à la valeur d'un texte normal, 700, à un texte en gras et 900, à un texte en très gras ou "bolder" (plus gras). La valeur se termine toujours par un point virgule ";" .

Pour conclure cette partie, l'on peut dire que le css est organisé de déclarations, ces déclarations contiennent une propriété, à qui on a donné une valeur et le tout se rattache à un élément ou groupe d'éléments html, le sélecteur, mais que sont les sélecteurs ?.
