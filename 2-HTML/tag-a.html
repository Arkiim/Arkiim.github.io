Partie 2 : HTML

La balise <a>

Comme expliqué dans la partie Histoire, la balise <a> est le nerf du Web, l'unique moyen de trouver son chemin jusqu'a la prochaine page ou jusqu'au prochain site. Cependant il existe différentes de manière de lier les pages entre elles, différents types de chemin, mais avant d'expliquer ceci, il faut d'abord comprendre de quoi est constitué l'élément <a>.

Représentation de la balise <a> utilisée pour lier cette image

Avec "a" pour "anchor" (ancre en anglais), l'élément <a> est tel une ancre qui va venir se loger et lier fermement deux site ou partie de site. Il composé exactement comme l'élément <img>, la balise <a> avec dedans un attribut qui lie encore vers quelque part, "href" pour "hypertext reference".

Comme dit précédemment dans la partie HTML, le site se comporte comme un gros dossier. Si la navigateur lit, affiche un des fichier html, il est aussi courant de tout les autres fichier et dossier qui sont dans le même dossier. Par exemple, tout les fichiers html de ce site sont rangé dans le même dossier et les images sont rangées au même endroit regroupées dans un dossier nommé "IMGs", actuellement le navigateur affiche cette page, mais il connait l'existence de tout les autres fichiers du site et pour pouvoir y accéder, il faut juste lui donner le "chemin relatif" jusqu'à eux.

Le chemin relatif est le chemin dont le point de départ est l'endroit où l'on se trouve actuellement, l'emplacement du fichier que l'on consulte. Il s'écrit comme sur la photo : chaque chemin relatif commence par "./" ces deux symboles annonce que l'on commence à chercher depuis notre position, que l'on utlise un chemin relatif. Ensuite la suite du chemin est exactement la même que n'importe quel chemin de fichier qu'on aurait cherché dans l'explorateur de fichier de l'ordinateur.

Exemple de chemin relatif avec les liens présent dans la navbar

Dans cette photo du code de la barre de navigation, on retrouve plein de chemin relatif qui pointent vers les différentes parties du site. Quand le navigateur verra, par exemple, le second lien, il cherchera dans le dossier actuel le fichier qui se nomme "Histoire.html". Les chemins relatifs ont plusieurs avantages non négligeables sur leurs pairs les chemins "absolues" (ceux que l'on l'habitude de voir) du type "https://www.sitesweb-dupasséaufutur.com/Histoire.html".

Premièrement ils sont beaucoup plus court ("./Histoire.html") et deuxièmement étant donné qu'ils font référence au dossier dans lequel on se trouve il n'y a aucun risque de se perdre, de perdre le fichier, qu'il ai disparu du site où on avait prit telle ou telle image ou encore que le navigateur n'arrive pas à le trouver.
