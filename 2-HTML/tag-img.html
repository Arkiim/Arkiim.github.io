Partie 2 : HTML

La balise <img>

Comme dit précédemment, les sites d'aujourd'hui regorgent d'images, mais comment sont-elles intégrées au site ?
Comme pour tout, avec un élément html : La balise <img>. Pour expliquer son fonctionnement, il va falloir prendre de l'avance sur la partie CSS et expliquer les notions de "attribute value" et "attribute name".

L'élément "img" s'écrit de cette manière, comme on peut le voir sur cette image qui est elle-même le code de l'image :

Représentation de la balise <img> utilisée pour insérer cette image

Le contenu de cet élément "img" contient ce qu'on appelle un "attribut". L'attribut est composé de son nom (attribute name) et de sa valeur (attribute value). Ici, l'attribut est tout ce qui est dans la balise et qui n'est pas rouge.
La partie verte représente le nom qui est : "src" (source), qui indique au navigateur qu'il doit rechercher quelque chose, en l'occurrence une image, à l'adresse fournie et cette adresse, en jaune, entre guillemets, est la valeur de l'attribut.

Si la valeur de l'attribut ne ressemble pas à une adresse du type : "https://www.sitesweb-dupasséaufutur.com/Histoire.html", c'est parce que l'adresse utilise ce qu'on appelle un "chemin relatif". Notion qui sera expliqué dans la partie suivante (même si les deux sont totalement équivalent dans leur fonction de lier).

En d'autres termes, pour que le navigateur affiche une image, il a besoin de l'ordre qui va initier cette demande : l'élément "img" et l'adresse, le "chemin" jusqu'à l'image, l'attribut "src".
