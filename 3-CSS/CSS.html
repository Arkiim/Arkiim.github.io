Partie 3 : CSS

A quoi sert le CSS ?

Le CSS sert à structurer les pages, à ajouter des "styles", des "règles" aux éléments html pour leurs donner une forme, une structure. Et ce, grâce à des propriétés CSS du type : aligner le texte à droite, au milieu, agrandir les marges, ajouter des espaces, mettre des éléments en colonnes...

Le CSS permet aussi de gérer l'aspect stylistique des éléments html, ( d'où le nom de Cascading Style Sheets ), il permet de modifier notamment, la couleur d'un titre, rajouter un background (image de fond), change la font (police d'écriture), aggrandir les éléments ou les raccourcir pour pouvoir les aligner par exemple, modifier la taille d'une image, sa position, souligner, ajouter une bordure...

Comparaison entre, la page actuelle avec CSS (à gauche) et sans CSS (à droite)

L'utilisation du CSS, pourrait être comparé avec ce que l'on fait d'ordinaire avec un logiciel de traitement de texte de type Libre Office Writer ou même avec un powerpoint, tout ce qui n'est pas du rajout de contenu, tout ce qui n'est pas purement de l'écriture dans ces types de logiciels, on le retrouve dans le CSS.

Cependant, même si coder du CSS est bien plus long et contient bien plus de risque de bugger et de ne pas fonctionner, il permet une liberté beaucoup plus grande, offre plus de possibilités et un meilleur contrôle.

Lier un fichier CSS

Etant donnée que le CSS est écrit dans un fichier à part, il faut pouvoir le rattacher aux fichiers html, sans quoi, le site ressemblera à l'image vu précédemment. Pour ce faire il faut utiliser une propriété html : l'élément <link>.
L'élément <link> est, à l'instar de l'hyperlien classique, composé d'attributs, trois pour être précis.

Photo du l'élément <link> liant le fichier css "TM-CSS.css", au fichier html

Le premier attribut est l'attribut "rel", pour "relation", qui spécifie la relation entre les 2 fichiers, ici il précise que le fichier lié est une feuille du style (stylesheet) du fichier de base.
Le second est l'attribut "type" qui définit le type du fichier et le troisième, le "href" avec, comme d'habitude, un chemin relatif menant jusqu'au fichier css "TM-CSS.css". Ainsi toutes les règles et propriétés css écrites sur ce fichier faisant référence aux éléments html de ce site, seront effectives.

Une fois le lien établi, l'on peut commencer à styliser le site en suivant la structure spécifique des déclarations CSS.

Box Model

La "box model" est sans aucun doute, le concept le plus important de tout CSS. Chaque élément html est positionné sur le site en fonction de sa box model. Lorsque l'on regarde cette page on pourrait croire qu'il y a des espaces, des trous, mais non, c'est en fait les différents parties de ces "boîte" qui s'aditionnent pour donner ce et cet impression d'espace.

Représentation de la box modele d'un élément html [28]

On a donc le "content", (contenu) qui est le contour que l'on voit à proprement parler, ici c'est la délimitation de la photo, bref l'idée de base que l'on pourrait avoir. Ensuite après le content on a le "padding", le padding est l'espacement entre le content et la "border" (bordure). La border pourrait s'assimiler à la "fin" de l'élément.

Enfin dernèrement on a la "margin", qui est l'espacement entre deux éléments, mais devant la border contrairement au padding qui lui est dans l'élément et derrière la border. Si elle n'est pas "matérialisé" manuellement avec du CSS la border ne se voit et alors on n'arrive plus à faire la distinction entre padding et border.

Comme une action vaut plus que mille mots je vous propose, pour la fin de cette partie, d'utiliser directement l'outil de développement de Google Chrome, afin de pouvoir vous représenter plus facilement ce box-model (clic droit+inspecter sur un élément). Si vous passez votre souris sur la balise <p> vous verrez trois rectangle s'afficher, le rectangle bleu représente le "content", le rectangle vert, le "padding", et le rectangle orange, "la margin".
